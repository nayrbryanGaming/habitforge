# Security Policy

## Supported Versions

We currently provide security updates for the following versions of HabitForge:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take the security of HabitForge seriously. If you believe you have found a security vulnerability, please report it to us responsibly.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please send an email to security@habitforge.app. Please include:

- A description of the issue
- Steps to reproduce the vulnerability
- Any potential impact

We will acknowledge receipt of your report within 48 hours and provide a timeline for resolution.

## Data Safety Commitment

HabitForge is built with a "Privacy First" mindset. All user data is isolated at the database level, and we perform regular audits of our Firebase Security Rules to ensure zero unauthorized access.

Our automated account deletion flow ensures that when a user requests deletion, every trace of their data is purged from our production environment.
